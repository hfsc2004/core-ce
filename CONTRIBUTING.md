# Contributing

## Scope
This repository hosts the Core - Community Edition code line for PSF Core.

## Before You Open a PR
1. Create a feature/fix branch from `master`.
2. Keep changes focused and reversible.
3. Run relevant checks/tests for changed areas.
4. Update docs when behavior or interfaces change.

## Commit Guidelines
- Use clear subject lines (example: `feat: add compliance evidence manager UI`).
- Include rationale in commit body for non-trivial changes.

## Licensing
By contributing, you agree that your contributions are licensed under the
repository license (`Apache-2.0`).

## Security
Do not disclose vulnerabilities in public issues. See `SECURITY.md`.

## Commercial/Private Boundaries
- Public/open components belong in this repo.
- Commercial/private modules and deployment artifacts should not be added here.
- If unsure, open an issue first and ask for boundary clarification.
