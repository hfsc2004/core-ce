# IP Ownership and Attribution Policy

Last updated: February 14, 2026

## Purpose
This document clarifies what Pseudo SF (PSF) owns in this project and what remains owned by third parties.

## Ownership Statement
PSF claims ownership of original work product created for this software project, including:
- Application architecture and pipeline design
- Original source code and modules authored by PSF
- Product integrations, orchestration logic, and automation
- UI/UX flows, screens, and interaction designs authored by PSF
- Project documentation authored by PSF
- Build, packaging, and distribution workflows authored by PSF

## Third-Party Components
Third-party components remain the property of their respective owners. This includes:
- Foundation models and quantized model artifacts
- External binaries and runtimes
- Open-source libraries and dependencies
- Vendor assets and third-party notices

Use, redistribution, and modification rights for third-party components are governed by their original licenses.

## Mixed/Modified Work
When PSF modifies third-party code or assets:
- Original third-party notices and license terms must be preserved.
- PSF may claim ownership only over the new modifications, not the underlying third-party work.
- The resulting file/component should be marked as "Mixed" in the provenance matrix.

## Attribution Requirements
Release artifacts should include:
- `LICENSE.txt`
- `ATTRIBUTIONS.md`
- Third-party notices and upstream license files as required
- `ProvenanceMatrix.md` with current ownership classification

## Operational Rule
No release should remove, obscure, or replace required third-party license notices.

## Scope Notes
This policy is operational guidance for this repository and does not replace legal counsel.
